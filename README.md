# Django Todo Application

## Objective
To build a robust and user-friendly Todo application using Django. The application aims to provide a seamless experience for users to manage their daily tasks efficiently, complete with organization features and user authentication.

## Steps Performed
Throughout the development of this project, several key milestones were achieved:
1. **Initial Setup and Exploration**: Explored the foundational Django configuration and set up the `tasks` app.
2. **Template Creation & Bug Fixes**: Created and debugged HTML templates (e.g. `index.html`) to ensure correct rendering.
3. **Backend Logic Validation**: Resolved Python import issues in `models.py` and `views.py` to correctly manage dependencies and avoid circular imports.
4. **Task Management Functionality**: Implemented and refined full CRUD operations, specifically focusing on building custom edit and delete capabilities.
5. **UI Enhancements**: Restructured the user interface to visually separate pending tasks from completed tasks, making it much easier to track progress.
6. **Authentication System**: Integrated a simple login system allowing users to register, log in, and securely access their tasks.

## Tools Used
- **Programming Language**: Python
- **Web Framework**: Django
- **Frontend**: HTML, CSS
- **Database**: SQLite (default Django database)
- **Environment**: Virtual Environment (`.venv`)

## Outcome
A fully operational, secure Todo application where users can register/login to view their personalized task list. The application features a clean interface that separates pending and completed tasks, and allows users to easily add, edit, lock, unlock, and delete tasks with confidence.
